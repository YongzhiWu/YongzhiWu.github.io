基于Zybo平台的微信智能处理系统设计

发表于 2018-07-08

一、引言

1.1 设计目的

  硬件课程设计是华中科技大学电子信息与通信学院相关专业本科学生必修的实践性教学环节。硬件课程设计的目的在于使学生通过硬件课程设计教学环节较系统地完成电子系统设计从选题、方案论证、电路设计、电路实现、装配调试、系统测试、总结报告等基本过程，加深对模拟电路、数字逻辑电路、通信电子电路、微机原理等相关课程理论知识的分析理解。引导学生把原理分析与工程设计实现相结合，掌握实际电子与通信系统设计的基本方法和一般规则，提高综合应用所学理论知识的能力，培养学生的创新思维和实践能力，为后续专业课程的学习打下坚固的基础。

  本课题期望了解利用各种传感器实现智能自动控制与信息处理或数据通信的基本原理，掌握其中一种方案的设计实现方式与技能，完成小组课程设计，为后续课程学习和工作打下基础。

1.2 设计背景

  本课题应用在智能家居方面，使用ZYBO板和腾讯公司的微信，开发一个智能养花系统，为一些由于其他原因不能回家，但又非常喜欢养花的人，能够在外面通过手机微信随时随地地观察家里面花地生长状况。考虑到影响花生长地因素有湿度、温度、光照等，我们在ZYBO板周围加入这些传感器，根据这些传给用户传感器的信息我们还能执行像浇水、关窗这样的操作；为了让养花者能够随时随地地方便地观看花地生长状况，还可以加入一个摄像头，这样养花者就可以通过微信看到花的图片。

1.3 课程设计调研

1.3.1 Zybo开发板

  ZYBO（Zynq Board）是一款资源丰富且易用的嵌入式软件及数字电路入门级开发平台，该平台主芯片为Xilinx Zynq-7000系列中的最小型号Z-7010。Z7010基于Xilinx全可编程片上系统框架结构（AP SoC）将双核Cortex-A9 ARM处理器与Xilinx系列FPGA紧紧集成在同一芯片上。ZYBO平台上集成有丰富的多媒体外设接口，强大的Z-7010芯片支持完整的系统设计。

  ZYBO支持3种默认编程模式，分别是：

  • JTAG编程模式：上电启动时，默认从PROG-UART USB接口编程。
  • QSPI编程模式：上电启动时，默认从板载QSPI Flash中读取配置文件来编程。
  • SD卡编程模式：上电启动时，默认从SD卡中读取配置文件来编程。

1.3.2 基于Zybo的嵌入式Linux系统

  在Zybo开发板上编写Linux应用程序，需要在Ubuntu系统上使用Vivado交叉编译工具，定制硬件模块，编译Linux核，将文件写入SD卡中，在SD卡编程模式下启动Linux系统。

1.3.3 微信订阅号

  微信订阅号为媒体和个人提供一种新的信息传播方式，主要功能是在微信侧给用户传达资讯（功能类似报纸杂志，提供新闻信息或娱乐趣事），它适用于个人、媒体、企业、政府或其他组织，它可以实现自动回复、自定义菜单等功能，用户可通过发送关键词使得服务器做出相应的响应，订阅号（认证用户、非认证用户）1天内可群发1条消息。

1.3.4 Apache服务器

  Apache是世界使用排名第一的Web服务器软件。它可以运行在几乎所有广泛使用的计算机平台上，由于其跨平台和安全性被广泛使用，是最流行的Web服务器端软件之一。经过快速安装、配置后，即可启用Apache服务器。

二、基础实验

  硬件课程设计分为基础实验和自选实验两部分，我们小组选择的都是基于Zybo平台的实验，基础实验部分由4个实验组成，为自选实验部分提供了基础。基础实验我们参考了老师给的实验文档。

2.1 使用Vivado创建嵌入式系统

实验目标

  • 创建Zynq系统Vivado项目文件；
  • 使用IP Integrator创建硬件系统；
  • 使用SDK创建标准内存测试项目；
  • 运行测试应用程序。

实验步骤

  1. 使用Vivado软件创建新项目；
  2. 使用IP Integrator创建处理系统；
  3. 生成顶层HDL，启动SDK导出硬件；
  4. 在SDK中创建内存测试程序；
  5. 在Zybo开发板上测试硬件。

问题说明

  该实验需要提前准备好Zybo开发板资源文件，开发板在JTAG编程模式下启动。

2.2 基于Zybo开发板设计IP核

实验目标

  • 使用IP Packager创建自定义外设；
  • 修改IP功能。

实验步骤

  1. 使用提供的定制IP创建自定义IP；
  2. 运行创建和打包IP向导
  3. 实现LED接口；
  4. 打包IP核。

问题说明

  新建IP时，需要注意Part Number为xc7z010iclg400-1L。在打包IP时，选择Customization GUI，需要添加参数LED_WIDTH，能够在面板中看到Led Width输入框才表明添加成功。

2.3 基于Zybo开发板实现IP核

实验目标

  • 添加自定义IP核到设计中；
  • 添加针位置约束；
  • 将块内存添加到系统。

实验步骤

  1. 将自定义的IP核添加到IP库中；
  2. 完成内部和外部端口的连接；
  3. 将BRAM添加到设计中；
  4. 添加提供的lab3_*.xdc文件。

问题说明

  如果没注意到基础实验2中的LED_WIDTH问题，在基础实验3中的Re-customize IP窗口就不能看到”LED_WIDTH”。

2.4 基于Zybo开发板编写应用软件

实验目标

  • 在SDK中编写基本应用程序；
  • 生成elf可执行文件;
  • 将比特流和应用程序下载到Zybo开发板上。

实验步骤

  1. 打开基础实验3完成的项目；
  2. 导入到SDK中，在SDK中创建新工程；
  3. 分析组装对象文件；
  4. 在硬件上验证功能。

问题说明

  观察Zybo开发板LED灯变化，发现根据4个开关对应的4位二进制编码不同，LED等变化的快慢不同。

三、自选实验

  参考老师发的智能浇花系统文档，我们预期的实验课题为基于Zybo平台的智能处理系统——智能浇花系统，最终我们实现了微信平台的搭建。

3.1 构思

  智能养花系统需要实现的功能有：

  • 基于微信平台的智能交互系统，能够自动回复用户消息，将用户指令发送给服务器，服务器进行响应。
  • 拍照记录植物状态，存储并发送照片给用户。
  • 获取植物所处位置的温度、湿度、光照强度等数据。
  • 执行用户命令，完成浇水。
  • 记录植物信息。

  因此，该系统需要的模块包括微信平台模块、摄像头模块、传感器模块、电机模块、数据存储模块。

3.2 设计

根据系统模块，系统组成框图如图所示

  系统运行逻辑状态框图如下所示

各模块设计功能如下：

  • 微信平台:与用户进行交互，自动回复用户消息。
  • 摄像头模块：摄像头每隔一段时间就会照下花的片。当用户想看，可以通过微信发送相应的命令，这个时候ZYBO板就会把花的照片通过微信发送给用户。
  • 传感器模块：包括湿度、温度、光照传感器，由pmod外设组成，分别获得花的土壤湿度、环境温度、环境光照强度，反馈给用户，让用户决定是否浇水。
  • 电机模块：一个小水泵，通过一个继电器控制，是否给花浇水。
  • 数据存储模块：分为服务器数据库和云存储两部分，前者存储服务器信息，后者通过云服务中的云存储API实现。每隔一段时间会自动将摄像头照下的花图片上传到云端，记录整个生长过程，以后用户可方便地重新查看花的生长过程。

3.3 实现

Ubuntu系统及SDK的安装

  我们采用双系统，在计算机上安装Linux的Ubuntu系统。参考老师分享文档 《Guidelines on VirtualBox Ubuntu and Xilinx SDK rev3-1-2018》以及百度经验制作安装U盘 和 双系统安装完成了Ubuntu系统以及SDK的安装。

  遇到的问题：双系统安装之后发现空间不足，无法安装SDK，然后我们重装系统，重新分配更大的内存，该问题得到解决。

嵌入式Linux系统的编译

实验步骤：

  1. 硬件自定义，使用Vivado实现自定义外设；
  2. 在Ubuntu系统下编译U-BOOT；
  3. 生成BOOT.BIN文件；
  4. 使用编译工具编译Linux内核；
  5. 使用预生成文件系统测试内核映像；
  6. 修改设备树，完成内核驱动，在SD卡编程模式下启动；
  7. 编写应用程序，进行测试。

  在做完交叉编译实验后，我们直接参考了ZYBO入门指导手册，将嵌入式系统Linaro移植到开发板上，在SD卡编程模式下启动。其中需要的准备工作可以参考ZYBO入门指导手册（一）附录2。

  我们遇到的问题：在TF卡中装Linaro系统时，TF卡无法挂载，我们通过自己的尝试，找到了解决方法：不要先分好区之后一个一个区挂载，而是分一个区，挂载一次，然后再分之后的区，再进行挂载。经实际检验，该方法具有可重复性。

四、评价总结

4.1 作品评价

  我们预期的目标本来是实现完整的智能浇花系统，但是由于时间的限制，最终我们只完成了微信平台服务器的搭建，可以实现自动回复用户的消息，但是其他模块并未完成，包括传感器模块、摄像头模块、数据存储模块，尤其是传感器模块和摄像头模块都需要通过自定义硬件来完成设计，需要再次编译。不过从时间上来看，我们能够在较短的时间内完成微信平台的搭建也是很不容易的。实际上，我们还希望能做出一些创新，比如讲微信订阅号改为微信小程序，相信如果有足够的时间我们还是能够实现完整的系统的。

4.2 课程总结

  本课程设计目标在于实现一个通信系统，不论是使用PlutoSDR平台完成一个收发系统，还是基于Zybo平台实现智能处理系统，都是在已经完成的课程基础上实现课程设计，让我们具有设计一个通信系统的能力。课程内容涉及到的知识包括智能家居应用需求，Xilinx Zynq系统设计和实现，同时我们学会了使用协同工作工具Redmine和文档版本控制工具SVN。课程设计要求我们获取文献资料，加强实践，学会沟通合作，掌握软硬件协作的方法。

4.3 个人感想

  在课程设计中，老师要求我们使用协作工具，在进一步实践中，我们认识到了使用Redmine和SVN的重要性。使用协同工作工具Redmine，能够有效地对我们的项目设计进行规划，每个成员可以及时在网上更新自己的任务，记录工作日志，有利于项目的开发和维护。与Git相似，SVN也是一种文档版本控制工具，每个成员都可以在个人电脑上有一个工作目录，同时在远程有一个公共目录，通过update和commit操作就可以完成工作空间文件的更新，提高了工作的效率。

  除了上述工具，本次课程设计，我们还学到了很多东西。这次课程设计，让我们接触到了嵌入式系统开发，我们对硬件层有了更多更深的接触，我们需要自己来编译内核，制作启动分区，让我们对嵌入式开发有了更深的理解。

  还有，通过在Linux上的操作，我们对Linux有了更多的认识，对Linux系统更加熟悉，对Linux系统下部署服务器有了一定的了解。

  不过遗憾的是，由于时间的限制，我们并未能完成整个智能浇花系统的开发，期望以后有机会能够实现一个更加完整的系统。

  首先感谢黑晓军老师给我们小组提供的帮助，老师给我们提供了很多的文档资料，解决了我们很多的困惑，让我们学会使用了很多有用的工具，使得我们完成了硬件课程设计。也感谢助教为我们提供的帮助，给我们提供了开发板以及需要资源文件。

  同时，感谢其他同学的支援，当我们遇到相似的问题时，我们可以向他们寻求帮助。另外，感谢我们小组的成员，在课程设计中起到了很重要的作用，在遇到问题时我们共同努力解决。在我们的的齐心协力下，最终完成了本次课程设计。最后，感谢所有在课程设计期间给我们提供帮助的人。
